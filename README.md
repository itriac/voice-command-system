# 語音指令控制系統

透過網頁麥克風收音，將語音轉為文字，比對內建指令後以 TTS 朗讀動作並等待確認，根據語意決定執行或重試。

## 需求

- Google Chrome 或 Microsoft Edge（Firefox 不支援 Web Speech API）
- 麥克風裝置
- Node.js（用於本地伺服器）
- **網路連線**（語音辨識需傳送至 Google / Microsoft 雲端服務）

## 啟動方式

> **注意：** 請勿直接雙擊 `index.html`。瀏覽器對 `file://` 不會記憶麥克風授權，每次都會重新詢問。請改用本地伺服器。

**方法一：雙擊批次檔**

雙擊 `啟動伺服器.bat`，瀏覽器會自動開啟 `http://localhost:3000`。

**方法二：終端機**

```bash
cd voice-command-system
node server.js
```

伺服器啟動後，在 Chrome / Edge 允許麥克風一次即會記住授權。按 `Ctrl+C` 停止伺服器。

| 紫色（閃爍） | 等待確認回覆 |

## 使用方式

1. 啟動伺服器，瀏覽器開啟 `http://localhost:3000`
2. 首次使用時允許瀏覽器存取麥克風
3. 按下麥克風按鈕（或按**空白鍵**）開始錄音
4. 說出指令，例如：「執行通感融合網元驗證」
5. 系統比對關鍵字後，TTS 朗讀：「執行通感融合網元驗證腳本，請回覆是否正確」
6. 系統自動開始聆聽確認回覆：
   - **正面語意** → 全螢幕顯示「動作執行中」＋ API 路徑＋進度條，完成後記錄至歷史
   - **負面語意** → TTS 朗讀「請重新提供命令」，回到待機狀態
   - **無法辨識** → TTS 詢問「請說正確或錯誤」，重新聆聽
   - **按麥克風按鈕**（或**空白鍵**）→ 中止確認流程，直接回到待機狀態

## 流程圖

```
按麥克風
    ↓
語音辨識（STT）→ 文字顯示於畫面
    ↓
指令關鍵字比對
    ├─ 無匹配 → TTS「無法辨識指令，請重新提供命令」→ 回到待機
    └─ 有匹配 → 顯示識別動作
                    ↓
               TTS 朗讀動作 + 「請回覆是否正確」
                    ↓
               自動開始聆聽回覆
                    ├─ 正面語意 → 全螢幕「動作執行中」+ API 路徑 + 進度條 → 記錄至歷史 → 回到待機
                    ├─ 負面語意 → TTS「請重新提供命令」→ 回到待機
                    ├─ 不明確  → TTS「請說正確或錯誤」→ 重新聆聽
                    └─ 按麥克風（取消）→ 中止確認，直接回到待機
```

## 內建指令對應

| 觸發關鍵字 | 執行動作 | API 路徑 |
|-----------|----------|----------|
| 通感融合、網元驗證、網元、驗證 | 執行通感融合網元驗證腳本 | `POST /api/v1/convergence/validate` |
| 健康檢查、系統健康、狀態檢查 | 執行系統健康狀態檢查 | `GET /api/v1/health` |

> 若語音不符合任何關鍵字，系統直接拒絕並要求重新輸入，不會進入確認流程。

## 正負面語意關鍵字

| 類型 | 關鍵字 |
|------|--------|
| 正面（確認執行） | 正確、對、好、是、確認、執行、沒錯、可以、yes、ok、對的、就這樣 |
| 負面（取消重試） | 錯誤、錯、不、否、取消、停止、重來、不對、不是、no、不行、不要 |

## 語音技術說明

### 語音辨識（STT）

使用瀏覽器原生 `window.SpeechRecognition`（Chrome/Edge 為 `webkitSpeechRecognition`）。雖為瀏覽器 API，但底層實作依賴雲端辨識服務，**需要網路連線**：

| 瀏覽器 | 音訊傳送目標 |
|--------|-------------|
| Chrome | Google Speech API |
| Edge | Microsoft Azure Speech |

斷網後語音辨識將失效，這是 Web Speech API 的平台限制，無法在純離線環境下使用。

| 參數 | 值 | 說明 |
|------|----|------|
| `lang` | `zh-TW` | 辨識語言設定為繁體中文 |
| `continuous` | `false` | 偵測到語音停頓後自動結束，避免無限錄音 |
| `interimResults` | `true` | 即時回傳暫存辨識結果，顯示為灰斜體 |
| `maxAlternatives` | `3` | 最多取 3 個候選結果，以第一個為準 |

辨識到最終結果後，以**子字串比對**方式掃描 `COMMANDS` 陣列的 `keywords`，第一筆命中者為匹配指令。

### 語音合成（TTS）

使用瀏覽器原生 `window.speechSynthesis`，語音偏好順序：`zh-TW` → `zh-Hant` → `zh-HK` → 任意中文聲音。

| 參數 | 值 |
|------|----|
| `lang` | `zh-TW` |
| `rate` | `0.92`（略慢於正常語速，提升清晰度） |
| `pitch` | `1.0` |

每次呼叫 `speak()` 前會先執行 `synth.cancel()` 清除佇列，確保不會與前一段播報重疊。

### 瀏覽器相容性

| 瀏覽器 | SpeechRecognition | SpeechSynthesis |
|--------|-------------------|-----------------|
| Chrome / Edge | ✅ | ✅ |
| Firefox | ❌ | ✅ |
| Safari | 部分支援 | ✅ |

兩個 API 均需在安全環境（`https://` 或 `localhost`）下才能取得麥克風授權，這也是不能直接開啟 `file://` 的原因。

## 擴充指令

在 `index.html` 的 `COMMANDS` 陣列中新增一筆記錄：

```javascript
// 不含 API 路徑
{ keywords: ['關鍵字A', '關鍵字B'], action: '執行對應動作名稱' },

// 含 API 路徑（執行時顯示於 overlay）
{ keywords: ['關鍵字A', '關鍵字B'], action: '執行對應動作名稱', api: 'POST /your/api/path' },
```
