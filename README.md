# 語音指令控制系統

透過網頁麥克風收音，將語音轉為文字，分析指令內容後以 TTS 朗讀下一步動作，並根據使用者的確認語意決定執行或重試。

## 需求

- Google Chrome 或 Microsoft Edge（Firefox 不支援 Web Speech API）
- 麥克風裝置
- Node.js（用於本地伺服器）

## 啟動方式

> **注意：** 請勿直接雙擊 `index.html` 開啟。瀏覽器對 `file://` 不會記憶麥克風授權，每次都會詢問。請改用本地伺服器。

**方法一：雙擊批次檔**

雙擊 `啟動伺服器.bat`，瀏覽器會自動開啟 `http://localhost:3000`。

**方法二：終端機**

```bash
cd voice-command-system
node server.js
```

伺服器啟動後，在 Chrome / Edge 允許麥克風一次，之後即會記住授權。按 `Ctrl+C` 停止伺服器。

## 使用方式

1. 啟動伺服器後，瀏覽器開啟 `http://localhost:3000`
2. 首次使用時允許瀏覽器存取麥克風
3. 按下麥克風按鈕（或按**空白鍵**）開始錄音
4. 說出指令，例如：「執行通感融合網元驗證腳本」
5. 系統將語音轉為文字，並以 TTS 朗讀：「執行通感融合網元驗證腳本，請回覆是否正確」
6. 說出確認或取消：
   - **正面語意**（正確、確認、是、可以…）→ 畫面顯示「動作執行中」並模擬進度
   - **負面語意**（錯誤、不、取消、重來…）→ TTS 朗讀「請重新提供命令」，回到待機狀態

## 流程圖

```
按麥克風
    ↓
語音辨識（STT）
    ↓
文字顯示於畫面
    ↓
指令關鍵字比對
    ↓
TTS 朗讀動作 + 「請回覆是否正確」
    ↓
自動開始聆聽回覆
    ├─ 正面語意 → 顯示「動作執行中」+ 進度條 → 記錄至歷史
    └─ 負面語意 → TTS「請重新提供命令」→ 回到待機
```

## 內建指令對應

| 觸發關鍵字 | 執行動作 |
|-----------|----------|
| 通感融合、網元驗證、網元 | 執行通感融合網元驗證腳本 |
| 健康檢查、系統健康、狀態檢查 | 執行系統健康狀態檢查 |

若輸入的語音不符合任何關鍵字，系統會將原始語音內容作為動作名稱，仍走完確認流程。

## 擴充指令

在 `index.html` 的 `COMMANDS` 陣列中新增一筆記錄：

```javascript
{ keywords: ['關鍵字A', '關鍵字B'], action: '執行對應動作名稱' },
```

## 正負面語意關鍵字

| 類型 | 關鍵字 |
|------|--------|
| 正面（確認執行） | 正確、對、好、是、確認、執行、沒錯、可以、yes、ok |
| 負面（取消重試） | 錯誤、錯、不、否、取消、停止、重來、不對、不是、no、不行、不要 |
